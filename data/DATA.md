# CourtSightR data

## Analysis-ready file

- **`games_2018_19_courtsightr.csv`**  -  Player-game rows for the **2018-19 NBA season window** (2018-10-01 to 2019-06-30), in CourtSightR’s expected schema. Use this as `--input` for analysis and split-usefulness evaluation.

  Generated by: `Rscript scripts/build_clean_games_2018_19.R` (see script for optional path argument).

## Source dataset

- The clean file is built from a **large NBA player-game CSV** (one row per player per game). That file is **not** stored in the repo; it is kept in **`data-raw/games_raw.csv`** (and ignored by git). Place your downloaded copy there before running the build script.

## Why 2018-19

- **Full season** (no lockout or mid-season stop).
- **Pre-COVID**  -  avoids schedule/availability noise from 2019-20 onward.
- **Cleaner than 2019-20** (which was shortened and played in a bubble).
- The source dataset stops on **2020-12-31**; 2020-21 is incomplete, so 2018-19 is a defensible single-season slice for evaluation.

## Regular season vs playoffs

- The current filter uses only the game date window (**2018-10-01** to **2019-06-30**).
- The source file does **not** provide a playoff/regular-season flag.
- Therefore, this slice includes **both regular season and playoffs** when present in that date range.

## Limitations

- `home_away` is derived from Basketball Reference-style `GAME_ID` (`YYYYMMDD0XXX`, where `XXX` is the home-team code) plus a code-to-team mapping. Rows that cannot be resolved are set to NA. For the tested 2018-19 slice from the current raw file, unresolved rows were 0.
- **Team/opponent** in the source are **full names** (e.g. "Phoenix Suns", "Dallas Mavericks"). CourtSightR normalizes them to uppercase; use full name or normalized form (e.g. "PHOENIX SUNS") when passing `--entity` for team mode.
- No claim is made about completeness or accuracy of the upstream source; use for local analytics and split-usefulness evaluation only.

## Raw-to-clean column mapping

| CourtSightR column | Raw column   | Notes |
|--------------------|-------------|-------|
| game_date          | GAME_ID     | Parsed from path (e.g. `/boxscores/201812250LAL.html` → 2018-12-25). |
| player_name        | PLAYER      | Direct. |
| team               | TEAM        | Full name (e.g. "Phoenix Suns"); not abbreviated. |
| opponent           | OPPT        | Full name. |
| home_away          | GAME_ID + TEAM/OPPT | Derived by parsing home-team code from `GAME_ID` (`YYYYMMDD0XXX`), mapping to full team name, then comparing against row `TEAM`/`OPPT` (`TEAM` match = `home`, `OPPT` match = `away`). Unresolved rows are set to NA. |
| win_loss           | RESULT      | "W" → "win", "L" → "loss". |
| minutes            | MP          | Direct. |
| points             | PTS         | Direct. |
| rebounds           | TRB         | Total rebounds. |
| assists            | AST         | Direct. |
| fg_made            | FG          | Direct. |
| fg_attempts        | FGA         | Direct. |
| fg3_made           | FG3         | Direct. |
| fg3_attempts       | FG3A        | Direct. |
| ft_made            | FT          | Direct. |
| ft_attempts        | FTA         | Direct. |
| turnovers          | TOV         | Direct. |
| plus_minus         | PLUS_MINUS  | Direct. |

## Split-usefulness evaluation

- An **exploratory** evaluation of split usefulness (win_loss, home_away, opponent) on the 2018-19 clean file is in **reports/split_usefulness_2018_19.Rmd**, with results in **reports/tables/split_usefulness_summary.csv**. The 2018-19 slice includes regular season and playoffs; findings describe practical split usefulness, not causation.

## Expanding to multiple seasons (optional)

To build a file for **2017-18 and 2018-19** combined, edit `scripts/build_clean_games_2018_19.R`: set `SEASON_START <- as.Date("2017-10-01")`, keep `SEASON_END` as is, and set `OUT_PATH` to e.g. `data/games_2017_19_courtsightr.csv`. Then re-run the script.
